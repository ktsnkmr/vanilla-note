gatsby cleanしたあとにgit pushができない

およそ2分で読める内容

今回の内容－
Gatsby cleanのあと、git pushでエラー。その対処と今後の回避の方法は？ netlifyがgatsby buildしてくれる？ 私の方法は邪道だったのか。

今回の対処の方法

git push -f origin main

とりあえず、強制的な手段で復旧はした。
あまり褒められた手段ではなさそうだ。

もっとほかの最適解があるはず。

そもそも不具合の原因はといえば

OSのクリーンインストールがしたかった。
これにつきるのだが、gatsby buildで不明なエラーが出なければ、gatsby cleanは回避できた。はず？

たびたびクリーンインストールはしないが、build時のエラーは再度遭遇しても不思議ではない。
今後もgatsby cleanの必要な時があるだろう。
その時、どうすればよいか？を考えたい。

どうすればよいの？

おそらくこうだ。

gatsby cleanは、.cacheとpublicを削除するので、gatsby cleanはせずに、それぞれ手動で削除する。
特にpublicは.git以外を削除して、削除したことをgit commitする。
その後はいつものとおりに。

もっとよい方法（正解）がありそう。
手動っていう操作は極力なくしたほうが現代的ではある。

＜メモ＞
.gitを勝手に削除するなってこと。← 私のことです。

gatsbyサイトのルートでgit initするのが通常か？

今回の件で調べてみたところ、私の方法は邪道なのかも。
他のサンプルを拝見すると、gatsby-config.jsやsrcをpushされている。

git initするフォルダがgatsbyサイトのルートだ。
.gitignoreで管理する方法だ。
そして、buildに必要なファイルをpushして、netlifyがbuildしてくれる。
これが本流のようだ。なるほど。

一部に私と似た方法の方もいらっしゃったので、build済みのpublicの中身をpushする方法も、あながち的外れではなさそうだ。

＜メモ＞
.gitを削除してしまって、強引にgit initすれば、リモートとローカルがおかしくなって当たり前。
今後は気をつけたい。gitの海は広そう。。。。
